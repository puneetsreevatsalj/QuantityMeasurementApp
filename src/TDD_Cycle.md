# Test Driven Development (TDD)

This project follows the Test Driven Development approach.

## TDD Cycle

1. Write Test Case
2. Run the test → It fails
3. Write minimum code to pass the test
4. Run the test → It passes
5. Refactor the code

## Implementation in this Project

### UC1
- Wrote equality test first
- Implemented Feet class
- Verified using JUnit

### UC6

## Advantages

- High test coverage
- Cleaner design
- Easier debugging
- Safe refactoring
